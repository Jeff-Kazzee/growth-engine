# Growth Retro Prompt

You are running a retrospective on my growth system itself. The goal is not to review my progress (that's the review prompt) — it's to improve the advice-giving system by updating my `playbook.md`, which every future growth session must read and obey.

## My state

```text
[PASTE playbook.md, decision-log.md, AND RECENT REVIEW/RADAR/POSITIONING OUTPUTS HERE]
```

## Procedure

1. **Analyze outcomes from evidence.** Which past verdicts turned out right or wrong? Which commitments got done vs repeatedly killed — and what did the done ones have in common (size, type, framing)? Which public artifacts actually moved my goals? Which playbook directives earned their keep; which are stale or ignored?
2. **Interview me** with 3–5 pointed questions: What advice did I actually use? What did I ignore, and why? What felt like busywork? Where was the system too soft or too harsh? What should it stop doing?
3. **Rewrite the playbook.** Numbered, testable directives (e.g., "weekend-size commitments only"); confirmed patterns moved into What works / What doesn't with evidence and dates; close finished experiments with results, add 1–2 new ones with hypotheses and check-by dates; prune dead directives. Keep it under ~150 lines. Never rewrite more than a third of the directives in one retro — evolve, don't thrash.
4. **Report.** What the system learned, what changed, what experiments are running, when the next retro should be.

Rule: base changes on the logs and my own words, not vibes. If a directive keeps getting violated, ask whether the directive is wrong before assuming I am.
